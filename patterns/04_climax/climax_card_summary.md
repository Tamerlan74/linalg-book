# Суть

Кульминация главы оформлена в виде **визуальной карточки**, на которой видно: один объект — несколько следствий. Это и есть пик «вау-момента»: читатель буквально **видит** всю красоту в одном изображении.

Эталон в нашей книге — карточка определителя: одна величина (определитель) — четыре ответа (растяжение / ориентация / обратимость / решение СЛАУ), в трёх случаях (+, −, 0). 9 ячеек, одна картинка, **главное содержание главы** перед глазами.

# Когда применять

- Когда **главное открытие главы** имеет **множественные следствия**, которые можно показать параллельно.
- Когда есть **естественные подслучаи** (положительный / отрицательный / нулевой; внутри / на границе / снаружи).
- В **главах с явной кульминацией**, чтобы зафиксировать момент.

# Когда не применять

- Если открытие **единичное** (одна формула, без множественных следствий) — карточка будет искусственной.
- Если **нет естественного разбиения на случаи** — нечего класть в колонки.
- Если **тема техническая** и плохо визуализируется — лучше прозой.

# Технические требования к карточке

- **Не больше 3-4 колонок** (случаев). 5 — уже теряется.
- **Не больше 4-5 строк** (следствий). 6 — перегруз.
- **Цветовое кодирование** случаев (зелёный/оранжевый/красный для +/−/0). Усиливает мгновенное восприятие.
- **Подпись наверху** — что объединяет все случаи («Одно число — четыре ответа»).
