# Суть

**Жёсткое правило**: перед введением каждого нового математического понятия — **минимум одна бытовая аналогия**. Формула появляется последней.

Это **базовый паттерн** Thompson. В главе II «Calculus Made Easy» центральное понятие «разные степени малости» вводится не через формулы, а через 4 бытовые иллюстрации подряд:

1. Минуты в часе и секунды в минуте.
2. Фартинг по сравнению с фунтом.
3. Миллионер → секретарь → мальчик.
4. Блоха на блохе у блохи.

**И только после** четвёртой иллюстрации — формальное $(dx)^2$ пренебрежимо мало.

# Какие источники аналогий работают

| Сфера | Когда брать |
|---|---|
| Деньги | Масштаб, отношения, проценты, накопление. |
| Движение и путь | Векторы, направление, скорость, расстояние. |
| Кухня и рецепты | Композиция, последовательность операций, преобразования. |
| Дом, ремонт | Структура, конструкция, основы. |
| Время | Малость, порядок величин, последовательность. |
| Музыка | Гармония, паттерны, частоты. |

# Какие аналогии НЕ работают

- Из IT (хеш-таблицы, API) — наш читатель не обязательно технарь.
- Слишком специфические (бухгалтерия, юриспруденция) — узкий круг узнает.
- Слишком эпохальные (фартинги, сюртуки) — современный читатель не знает контекст.
- Политические или культурно-чувствительные — рискует устареть.

# Когда не применять

- В чисто технических разделах (доказательство леммы) — там аналогия отвлекает.
- В **связке с уже введённым понятием**, если читатель уже владеет аналогией из прошлой главы.
