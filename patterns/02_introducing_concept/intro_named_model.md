# Суть

Для **фундаментальных понятий** книги — давать **именованную визуальную модель** с собственным названием. У Blitzstein это «Pebble World» для пространства исходов. У нас это «матрица как инструкция преобразования», а позже — «композиция инструкций».

Имя модели становится **мнемонической опорой**, к которой можно возвращаться через сто страниц: «помните, у нас была композиция инструкций — а теперь...».

# Признаки хорошего имени модели

| Признак | Пример хороший | Пример плохой |
|---|---|---|
| Короткое | «инструкция преобразования» | «правило линейного отображения с матричной репрезентацией» |
| Конкретное | «композиция инструкций» | «общий формализм» |
| Визуальное | «параллелограмм-площадь определителя» | «двумерная характеристика» |
| Запоминающееся | «Pebble World» | «set of outcomes» |

# Когда применять

- Один-два раза на главу, для **самого важного** понятия.
- Особенно — для понятий, которые **повторяются в будущих главах** (например, «инструкция преобразования» используется во всех последующих главах книги).

# Когда не применять

- Для **производных или вторичных** понятий. Если каждому понятию давать имя — все имена обесцениваются.
- Если у понятия **нет естественной визуализации** — натянутая модель хуже, чем её отсутствие.

# Шаблон применения

> Будем называть это <короткое имя модели>. Каждый раз, когда у нас будет <ситуация>, мы будем мысленно представлять <визуальный образ>. Запомните это имя — мы к нему ещё вернёмся.

# Связка

`intro_analogy_first` + `intro_named_model` — сначала аналогия активирует интуицию, потом модель получает имя. Это сильная пара.

# Эволюция модели

Если модель введена в одной главе, в следующих её можно **расширить**: «Помните инструкции преобразования? Сейчас мы увидим, что некоторые инструкции бывают „хорошими“ — они...». Расширение модели — это паттерн `book_recurring_map` уровня книги.
