# Суть

После символьного вывода — **обязательная численная проверка** на конкретных цифрах. Берётся конкретный пример, повторяется вся выкладка на числах, показывается, что получается то же самое. Снимает скепсис вида «ну вы вывели какую-то формулу — а оно вообще работает?».

В главе IV Thompson выводит, что для $y = x^2$ производная равна $2x$. **И тут же**: *«Numerical example. Suppose x = 100... y = 10,000. Let x grow till it becomes 101...»*. Берёт конкретное число, повторяет всю выкладку на числах, показывает, что получается то же самое.

# Когда применять

- После **каждого нетривиального** символьного результата.
- Особенно — при первом введении формулы (когда читатель её ещё не присвоил).

# Когда не применять

- Для **тривиальных тождеств** ($a + b = b + a$).
- Когда символьный вывод **сам по себе** убедителен (через очевидную геометрию).

# Технические требования к цифрам

| Что | Как |
|---|---|
| Числа | Круглые (100, 1000, 0.1). Чтобы не отвлекать на счёт. |
| Количество примеров | 1-3. Один обязательно, дальше по необходимости. |
| Покрытие случаев | Если у формулы есть «крайние случаи» (нулевой определитель, отрицательный), показать по одному примеру каждого. |

# Главное правило: вычисления делает не LLM

**В нашем воркфлоу численная проверка генерируется не текстом, а вычисляется** — на этапе 3 пайплайна (Python+SymPy). LLM ставит маркер `[NUMERIC_CHECK: формула, переменные]`, дальше скрипт его раскрывает в конкретные числа.

Это **критический пункт** — у нас в главах 3 и 4 были ошибки именно потому, что LLM сама пыталась считать. Не должна.

# Шаблон

```
### Численные проверки

Проверим формулу на конкретных матрицах.

**Первая.** $A = [NUMERIC: A1]$. Определитель: $[NUMERIC: det_A1] = [NUMERIC: ad-bc_A1]$. <комментарий>.

**Вторая.** ...
```

В таком виде LLM может писать черновик; конкретные числа подставит вычислитель.
