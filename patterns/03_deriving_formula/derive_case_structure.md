# Суть

При выводе общего правила — сначала **3 частных случая по одной структуре**, потом обобщение. Не «вот формула, теперь примеры», а «вот пример, вот ещё, вот ещё — заметили закономерность? — вот формула».

В главе IV «Calculus Made Easy» Thompson так выводит правило дифференцирования: Case 1 — $y = x^2$. Case 2 — $y = x^3$. Case 3 — $y = x^4$. Та же структура, тот же ритм. К Case 3 читатель уже предсказывает, что получится. Общая формула выводится почти сама.

# Пример

При введении правила применения матрицы к вектору:

> **Случай 1.** Применим $M = \begin{pmatrix} 2 & 0 \\ 0 & 1 \end{pmatrix}$ к вектору ...
>
> **Случай 2.** Применим ту же матрицу к другому вектору...
>
> **Случай 3.** Другая матрица. Возьмём $A$...
>
> Заметили закономерность? **Каждая строка матрицы по отдельности «спаривается» с вектором** ...

# Когда применять

- Для **главных правил** главы.
- Когда у правила есть **варьируемый параметр**, и можно показать ту же процедуру с разными значениями.
- Для **читателя, который учится по образцу** — то есть для нашего читателя.

# Когда не применять

- Когда правило **уникально** и не имеет естественных подслучаев (например, формула $ad - bc$ — её не разнесёшь на case-ы).
- Когда case-ы будут **тривиально одинаковыми** — читатель заскучает.

# Технические требования к case-ам

- **Не меньше 3** — два частных случая не дают паттерна.
- **Не больше 4** — больше затягивает.
- **Одинаковая структура** — каждый case построен по одному шаблону: те же шаги, тот же ритм.
- **Постепенное усложнение** — Case 1 простейший, Case 3 чуть сложнее. Но не слишком.

# Шаблон

```
**Случай 1.** <простейший вариант>. <шаги>. Результат: <X>.

**Случай 2.** <вариант с одним отличием>. <те же шаги>. Результат: <Y>.

**Случай 3.** <вариант с другим параметром>. <те же шаги>. Результат: <Z>.

Заметили закономерность? <общее наблюдение>.

Формула:
$$<общая формула>$$
```
